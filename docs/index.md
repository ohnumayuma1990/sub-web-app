# 低コスト・収益化可能なWEBアプリ案 一覧

本ドキュメントでは、運用コストを最小限に抑えつつ、複数の収益源を確保できる8つのWEBアプリ案を紹介します。

## 企画一覧

| 案 | タイトル | 主な収益源 | 運用負荷 | 特徴 |
| :--- | :--- | :--- | :--- | :--- |
| [案1](01_specialized_calculator_tool.md) | 特化型計算ツール | 広告・アフィリエイト | 低 | メンテナンスが容易でSEOに強い |
| [案2](02_niche_resource_directory.md) | ニッチディレクトリ | アフィリエイト・広告 | 中 | 情報の集約自体に価値がある |
| [案3](03_micro_saas_utility.md) | マイクロSaaS | サブスクリプション | 高 | 安定した収益が見込める |
| [案4](04_community_knowledge_base.md) | ナレッジベース | 広告・寄付 | 低（中） | ユーザによるコンテンツ生成 |
| [案5](05_ai_learning_tool.md) | AI学習ツール | サブスク・課金 | 中（高） | パーソナライズされた体験 |
| [案6](06_niche_marketplace.md) | ニッチ市場 | 手数料 | 高 | 委託販売による高い信頼性 |
| [案7](07_realtime_monitor.md) | リアルタイム監視 | サブスクリプション | 中 | 即時性が価値を生む |
| [案8](08_expert_matching.md) | 専門家マッチング | 手数料 | 極高 | 人的対応による高満足度 |

## 共通のコスト削減施策
- **サーバレス / 静的ホスティング**: Vercel, Cloudflare Pages, Firebase 等を活用し、インフラ維持費を無料〜低額に抑える。
- **OSS / ヘッドレスCMS**: 既存の強力なツールを組み合わせ、開発・管理コストを削減。
- **自動化**: GitHub Actionsなどを用いたビルド・デプロイ・定期的なデータ取得の自動化。

## 運用の考え方
各案とも、最初は「一人で回せる」規模からスタートし、収益が発生し始めてからコンテンツの拡充や機能追加に投資する構成を想定しています。
詳細は各ドキュメントを参照してください。
