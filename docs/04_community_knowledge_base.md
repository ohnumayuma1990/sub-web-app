# 案4：コミュニティ主導型ナレッジベース（レトロゲーム攻略・情報Wiki）

## 概要
特定のニッチなジャンル（例：80-90年代のレトロゲーム）に特化した、ユーザ参加型の情報共有プラットフォーム。攻略情報だけでなく、当時の関連資料や周辺機器の情報も集約します。

## 収益化方法
- **広告掲載**: ターゲット層（30代後半〜50代）にマッチしたレトロ系商品やホビー関連の広告。
- **アフィリエイト**: 中古ゲーム販売サイト（駿河屋、eBay等）、互換機、レトロゲーム復刻版（Switch Online等）の紹介。
- **寄付（Donation）**: Buy Me a Coffee や GitHub Sponsors を通じた、熱心なコミュニティメンバーからの支援。

## 維持コスト削減策
- **Wikiエンジンの活用**: MediaWikiなどの定評のあるOSSを活用するか、GitHub Pages/Actionを活用した「Git-based Wiki」構成にする。
- **CDNの活用**: 画像などの静的アセットをCloudflare等でキャッシュし、転送料金を抑える。
- **コミュニティモデレーション**: 熱心なユーザに管理者権限を付与し、コンテンツの整理や不適切投稿の監視をボランティアベースで依頼する。

## メリット・デメリット
### メリット
- ユーザがコンテンツを作成してくれるため、運営側のコンテンツ作成負荷が低い。
- ロングテールな検索流入が期待でき、息の長いサイトになる。

### デメリット
- 初期のコミュニティ形成が難しい（「ニワトリと卵」問題）。
- 荒らし対策や著作権への配慮が必要。

## 想定運用費
- **ドメイン費用**: 年間 約1,500円
- **サーバ費用**: 月額 500円 〜 1,000円（VPSまたはマネージドな軽量サーバ）
- **合計**: 月額 約650円 〜 1,200円

## 運用負荷（人的リソース）
- **初期構築**: 1〜2週間（Wikiのセットアップと基本構造の作成）。
- **定常運用**: 週1〜2時間のモデレーション、システムのバックアップ確認。
- **コミュニティ形成**: 初期段階での積極的な投稿とSNSでの拡散。
