# 案3：マイクロSaaSユーティリティ（SNS画像自動生成・管理ツール）

## 概要
ブログ記事のタイトルやアイキャッチ画像を、テンプレートに基づいて自動生成し、主要なSNS（Twitter, Instagram等）への投稿用フォーマットに変換・保存するツールです。

## 収益化方法
- **サブスクリプション**: 月額数百円〜の低価格プラン。無料枠（作成枚数制限）を設け、プレミアム機能として「テンプレート無制限」「ロゴ非表示」「高画質書き出し」などを提供。
- **都度課金（Pay-as-you-go）**: クレジット制で購入した分だけ画像を生成できる仕組み。

## 維持コスト削減策
- **Firebase / Supabase**: 認証、データベース、ストレージを統合管理し、初期費用と管理の手間を最小化。
- **Canvas API (Client-side)**: 画像生成処理を可能な限りクライアント（ブラウザ）側で行い、サーバ側のCPU負荷を削減。
- **OSSライブラリの活用**: Fabric.jsなどの画像編集ライブラリを活用し、開発コストを削減。

## メリット・デメリット
### メリット
- 継続的な利用が見込める（リピート性が高い）。
- 一度ツールを気に入ってもらえれば、サブスクリプションによる安定収益につながる。

### デメリット
- 画像ストレージのコストが増加する可能性がある。
- SNSのAPI仕様変更などの影響を受けやすい。
- ユーザサポートが発生する可能性がある。

## 想定運用費
- **ドメイン費用**: 年間 約1,500円
- **インフラ費用**: 月額 0円 〜 3,000円（ユーザ数に応じてスケール、Supabase等の無料枠から開始）
- **合計**: 月額 約130円 〜 3,200円

## 運用負荷（人的リソース）
- **初期構築**: 1ヶ月程度の開発。
- **定常運用**: 月数時間のサーバ監視、バグ対応。
- **ユーザサポート**: 週1〜2件程度の問い合わせ対応。
- **アップデート**: 新しいテンプレートの追加（月1回）。
